
Selected Semester Project Opportunity

Selected Project Title

Smart Assignment Reminder App

Target User

Who has the problem?

University students who manage multiple courses, assignments, quizzes, and examination schedules.

Problem Statement

What problem does the user experience?

University students often forget assignment deadlines, quiz dates, and project submission times because information is spread across different platforms such as learning management systems, class messaging groups, and handwritten notes. Missing deadlines can negatively affect academic performance and increase stress.

Current Alternative

How do users currently solve or manage this problem?

Students currently use notebook planners, phone calendars, alarm applications, sticky notes, spreadsheets, and messaging group reminders to track deadlines. However, these methods are often disconnected, require manual updates, and do not effectively prioritize urgent tasks.

Proposed IT Solution

Develop a mobile application that allows students to record assignments, organize tasks by course, receive automatic notifications before deadlines, and view all academic activities in a single dashboard.



Why This Is Feasible Without IoT or Advanced Cybersecurity

Explain why the team can build or simulate the MVP using available tools.

The project does not require physical devices, sensors, or IoT integration. All core features can be implemented using standard software technologies such as mobile frameworks, cloud databases, and notification services.

The MVP will use basic user authentication and data storage without handling sensitive financial, medical, or confidential information. Therefore, advanced cybersecurity mechanisms are unnecessary at the prototype stage.

The team can build and test the solution using commonly available tools such as Figma, Flutter, React Native, Firebase, and Google Sheets.

Possible MVP Type

Choose one MVP type.

Simple web app prototype

The MVP will include essential features only:

* User login
* Assignment creation
* Deadline management
* Notification simulation
* Dashboard for upcoming tasks

Initial Validation Plan for Lab 03

Who will the team ask?

The team will interview and survey 15-20 university students from different courses and academic years.

What evidence will the team collect?

* Survey responses about missed deadlines
* Frequency of current reminder tool usage
* Pain points with existing solutions
* User feedback on prototype screens and features

What question must be answered first?

Do university students experience enough difficulty managing assignment deadlines to regularly use a dedicated reminder application?
